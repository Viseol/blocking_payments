# Cтруктура базы данных
Система управления блокировками платежей для юридических лиц в Т-Банк.
## Описание: 
В данной базе данных хранится информация о клиентах и платежах, а также об их статусах.
## Инфомация о клиентах
### Название: clients
### Описание: В таблице хранится информация о клиентах и их статусе.
### Описание полей таблицы:
1. client_id - уникальные код клиента;
2. name_client - ФИО клиента;
3. INN - идентификационный номер налогоплательщика;
## Статусы платежей
### Название: payments_code
### Описание: Содержит информацию о статусах платежей.
### Описание полей таблицы: 
1. code_id - уникальный код статуса платежа (например, для успешного платежа это всегда 0);
2. code_name - короткое название (расшифровка) кода платежа. Например FRAUD/DATA_ERROR/SUCCESS/FOREIGN_CARD_BLOCK и так далее;
3. code_description - полное описание кода платежа. Например, это может быть 'Неверные введенные данные' или 'Невозможен перевод на иностранную банковскую карту' и др.;
4. category - FRAUD/OTHER - по этому столбцу как раз можно отпличать блокировку по мошенничеству от других причин.
## Платежи
### Название: payments
### Описание: Содержит информацию о платежах.
### Описание полей таблицы:
1. code_payment - уникальный код платежа;
2. client_id - ID клиента совершающего платеж;
3. code_id - код платежа;
4. amount - сумма платежа;
5. time_payment - время и дата платежа
6. blocked - платеж может либо заблокирован - true, либо успешным -false.
## История блокировок
### Название: block_history
### Описание: Содержит информацию о блокировках клиентов.
### Описание полей таблицы:
1. history_id - уникальный код блокировки;
2. client_id - ID клиента над которым совершается операция;
3. action - BLOCKED/UNBLOCKED;
4. code_id - ID кода по которому совершвется блокировка;
5. initiated_by - имя работника, которым был совершена блокировка/разблокировка;
6. initiated_at - время, когда была совершена блокировка/разблокировка.
