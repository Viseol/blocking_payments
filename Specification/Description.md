# Blocking Client API
## API предназначено для управления блокировками платежей клиентов — позволяет блокировать, разблокировать и проверять статус блокировки по клиентам.
## Описание:
## Blocking Client API предоставляет функциональность для систем, обрабатывающих клиентские платежи.
С его помощью можно:
1. Заблокировать все платежи клиента (например, по причине мошенничества или запроса клиента);
2. Разблокировать все платежи клиента;
3. Проверить текущий статус блокировок ( с помошью этого поинта можно узнать категорию блокировки - FRAUD/OTHER);
4. Узнать, находится ли клиент в заблокированном состоянии

## Endpoints:

### POST /Client/{client_id}/blockAll
### Назначение: заблокировать все платежи клиента.
### Параметры:
1. client_id — ID клиента;
2. code_id — код причины блокировки;
3. initiated_by — инициатор операции (например, логин сотрудника).
### Ответы:
1. 200 — платежи успешно заблокированы (BlockInfo);
2. 404 — клиент не найден;
3. 409 — некорректный код блокировки;

### POST /Client/{client_id}/unblockAll
### Назначение: разблокировать все платежи клиента.
### Параметры:
1. client_id (path, integer)
2. initiated_by (query, string)
## Ответы:
1. 200 — успешно разблокировано
2. 404 — клиент не найден

### GET /Client/{client_id}/getStatus
### Назначение: получить детальную информацию об активной блокировке клиента.
### Параметры:
1. client_id (path, integer)
### Ответы:
1. 200 — Информация о статусе блокировки получена (возвращает данные с деталями по блокировке)
2. 404 — клиент не найден
### Дополнительно:
С помошью данного endpont'а можно можно отличать блокировки мошенников от блокировок добропорядочных клиентов

### GET /Client/{client_id}/blockStatus
### Назначение: быстро проверить, находится ли клиент в статусе блокировки.
### Параметры:
1. client_id (path, integer)
###Ответы:
1. 200 — статус клиента получен (возвращает объект со свойством block_status (например, "BLOCKED" или "ACTIVE"))
2. 404 — клиент не найден

## Дополнительно:
Вышеперечисленные endpoints реализует лишь задачи из кейса, в случае необходимости их можно дополнить.
